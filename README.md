9. Thực hành các thao tác Git trên dự án RE Shopper Web UI
9.1. Thêm 1 người tham gia vào RE Shopper Repository mục Collaborators
9.2. Live pair-working demo với người tham gia về việc xử lý xung đột code:
9.2.1. Tạo nhánh mới (dev) từ nhánh main, thiết lập nhánh mặc định là dev
9.2.2. Mỗi người tạo một nhánh mới (feature) từ nhánh dev (dev tạo từ main)
9.2.3. Cả 2 cùng sửa tên dự án thành <Tên người tham gia> Shopper (tạo xung đột)
9.2.4. Người A đẩy code lên GitHub trước, tạo Pull Request thứ nhất
9.2.5. Gửi lên chat Zoom để nhờ Leader review, comment lên PR, approved PR
9.2.6. Người A merge code vào dev, thông báo cho người B pull code mới về
9.2.7. Người B pull code mới về sẽ gặp conflict với file code người B đã sửa
9.2.8. Người B sẽ pair-reviewing với người A, xử lý xung đột code để giữ code cả 2
9.2.9. Người B sẽ sửa tên dự án thành <tên người A + tên người B> Shopper
9.2.10. Xử lý xong xung đột code, người B tạo Merge Conflict và đẩy PR lên GitHub
=> Người B gửi link Pull Request lên chat Zoom nhờ Leader review (không bấm merge)
Làm tới đây thì người A và người B gửi link PR lên nhóm Zalo để hoàn thành Case Study
